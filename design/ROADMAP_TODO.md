# Roadmap Todo

本文记录 2026-06-26 讨论过、但尚未完整产品化的后续可拓展点。优先级仅表示当前判断，可按实际业务调整。

## P0 稳定性与运维

### 数据库迁移规范化

- 引入正式 migration 工具，例如 Flyway 或 Liquibase。
- 每次 Entity/表结构变更必须同时提交 migration SQL。
- `schema.sql` 只作为新库初始化，不作为已有库升级依据。
- 对已有库上线顺序固定为：先执行 migration，再发布依赖新字段的应用代码。
- 为 migration 增加回滚说明和上线验证 SQL。
- 参考：[DB_MIGRATION_MEMORY.md](./DB_MIGRATION_MEMORY.md)。

### 本地限流策略优化

当前 `api_key.rate_limit` 存在数据库中，适合按客户/Key 单独配置；但默认值仍硬编码在代码里。

待开发：

- 将新建 API Key 的默认限流值改为配置项，例如 `API_KEY_DEFAULT_RATE_LIMIT`。
- 管理后台支持调整单个 Key 的 `rateLimit`。
- 区分接口类型限流：
  - 模型/问答接口严格限流。
  - 文档列表、状态轮询等管理接口单独限流或降低计数权重。
  - 上传接口可设置独立限流。
- 本地 429 和上游云平台 429 在错误消息和日志中明确区分。

### RAG 索引队列产品化

当前已加入轻量队列和 embedding 节流，后续可继续增强：

- 页面展示待索引队列长度、当前处理文档、预计剩余时间。
- 支持暂停/恢复索引任务。
- 支持取消某个待处理文档。
- 失败任务支持重试。
- 队列状态持久化，避免应用重启后任务丢失。
- 大批量导入时按租户隔离队列，避免一个租户堵住全部索引任务。

## P1 RAG 与知识库能力

### 阿里百炼 RAG 深度对接

当前已支持通过 env 在本地 RAG 和百炼 Retrieve API 之间切换，但仍属于“检索接入”阶段。

待开发：

- 增加本地知识库和百炼 `indexId` 的映射关系。
- 支持每个知识库独立配置云端 RAG 索引，而不是全局一个 `RAG_BAILIAN_INDEX_ID`。
- 支持把上传到本系统的文档同步到百炼知识库。
- 支持百炼文档上传、删除、重建索引、同步状态查询。
- 云端 RAG 模式下，页面提示“当前检索来源为百炼云端索引”。
- 云端检索返回的来源 metadata 做更完整映射，例如文档名、页码、chunk id、原文链接。
- 本地/云端 RAG 做 AB 对比调试面板。

### 本地 RAG 质量继续优化

当前本地 RAG 已有 PDF 按页解析、混合检索、关键词加权、本地重排等基础能力，但对技术规格类 PDF 仍有提升空间。

待开发：

- 表格解析增强，避免 PDF 表格信息被打散。
- 支持按标题/章节结构切片。
- 支持父子切片：小 chunk 用于检索，大上下文用于生成。
- 支持 query 改写可视化，便于看模型把问题改成了什么。
- Debug 页面展示向量分、BM25 分、rerank 分、最终分。
- 建立一批固定评测问题，用于对比每次检索策略调整的效果。
- 对重复 chunk 或同页重复召回做去重，避免引用来源刷屏。

### 文档去重完善

当前新增文档会计算 SHA-256，并在同一知识库内跳过重复索引。

待开发：

- 为历史旧文档回填 `file_hash`。
- 提供“一键扫描重复文档”工具。
- 重复文档清理前展示保留项和待删项。
- 支持跨知识库重复检测，但不默认阻止跨知识库复用。
- GitHub 导入也加入 repo/branch/path 维度去重。

### 知识库文件管理增强

当前已支持文档多选和批量删除。

待开发：

- 支持按状态、类型、来源、时间筛选文档。
- 支持按文件名搜索。
- 支持批量重建索引。
- 支持批量导出文档元数据。
- 删除文档时同时展示将清理的向量数量。
- 正在索引中的文档支持取消任务后删除。

## P1 GitHub 仓库导入

当前 GitHub 导入已有基础链路：拉取公开仓库文件树、下载文本文件、拼接后切片入本地向量库。

待开发：

- 支持 `GITHUB_TOKEN`，提高 API 限额并支持私有仓库。
- 支持导入仓库子目录，例如 `/tree/main/docs`。
- 支持带斜杠的分支名，例如 `/tree/feature/rag-import`。
- 按文件独立建文档，而不是把整个仓库拼成一个大文档。
- 支持增量同步：只更新变更文件。
- 支持忽略规则配置，例如 `.gitignore`、自定义 include/exclude。
- 对超大仓库增加文件数、总大小、总字符数限制和预估提示。
- GitHub 导入也接入索引队列、任务进度、失败重试。
- 如果启用百炼 RAG，支持将 GitHub 内容同步到百炼索引。

## P1 售后问题收集知识库

目标：把零散、破碎的售后问题沉淀成可检索、可归档、可复用的知识库。

数据来源：

- 客服/售后聊天记录。
- 飞书群消息。
- 人工录入的问题片段。
- 文档、截图、日志片段。

待开发：

- 新建“售后问题收集知识库”类型。
- 支持原始问题片段导入，不要求一开始就是完整文档。
- 对问题自动抽取结构化字段：
  - 产品/型号
  - 问题现象
  - 触发条件
  - 初步原因
  - 解决方案
  - 状态：待确认、已解决、已归档
  - 来源：聊天、飞书、人工、工单
- 支持多条碎片自动聚合成一个问题卡片。
- 支持人工确认、编辑、合并、拆分问题卡片。
- 支持从问答结果中反向沉淀“已验证答案”。
- 后续对接飞书群机器人或飞书开放平台，自动采集指定群消息。
- 涉及隐私数据时，提供本地存储和云端存储开关，并明确数据流向。

## P3 数据隐私与云端能力边界

### 工作流与知识库多租户隔离

当前系统按单租户模式运行，部分工作流接口和前端请求仍默认或硬编码 `tenantId=1`。启用多租户前必须完成以下隔离改造：

- 后端只从已验证的 API Key / `RequestContext` 获取租户 ID，不信任客户端传入的 `X-Tenant-Id`。
- 工作流详情、更新、删除、发布、执行接口均按 `workflowId + tenantId` 校验归属。
- `ToolNodeExecutor`、`KnowledgeNodeExecutor` 和 `KnowledgeBaseTool` 使用知识库 ID 前，校验其属于当前租户。
- 工作流执行记录、节点执行记录和审批记录的查询、取消、审批操作按租户隔离。
- 百炼云端模式建立 `tenantId + knowledgeBaseId -> indexId` 映射，避免多个租户共享默认索引造成数据串读。
- 增加跨租户越权测试：篡改工作流 ID、执行 ID、知识库 ID 时必须返回 403/404，且不得发起检索或模型调用。

### 长期记忆与云端存储说明

后续需要在管理后台或文档中明确：

- 长期记忆是否启用。
- 记忆数据存储在本地还是阿里云端。
- 上传到云端的数据类型。
- 如何关闭云端记忆。
- 如何删除用户记忆。

### RAG 数据流向说明

当启用百炼 RAG 时，需要明确：

- 文档是否上传到百炼。
- 检索 query 是否发送到百炼。
- 返回内容是否落本地库。
- 本地 RAG 与云端 RAG 的数据边界。
