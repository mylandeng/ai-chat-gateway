# 代理池子系统设计文档

## 概述

代理池管理子系统，提供 IP 扫描、账号池管理、代理转发网关、数据看板四大核心功能。
作为独立子菜单嵌入现有管理后台，与知识库系统并列。

## 数据流

### 扫描流程
```
用户添加IP → 选择扫描脚本 → 提交扫描任务 → ProcessBuilder调用Python
→ 解析输出 → 匹配账号入库 → 健康检查
```

### 代理转发流程
```
客户端请求 → 网关Controller → 参数清洗 → 路由策略选择账号
→ WebClient转发 → SSE流式响应 → 异步记录日志
```

### 健康检查流程
```
定时触发/手动触发 → 遍历启用账号 → GET /models → 更新状态
→ 不健康时触发告警 → 日志+Webhook推送
```

## 扩展接口 (预留)

- `ProxyRoutingStrategy`: 负载均衡策略接口
  - 当前实现: `RandomWeightedStrategy` (加权随机)
  - 预留: RoundRobin, LeastConnection, Failover
- `AlertService`: 告警推送
  - 当前: 日志 + Webhook
  - 预留: 邮件, 钉钉/飞书/企微
- 网关路径 `/api/proxy/gateway/v1/*`: OpenAI 兼容格式

## 注意事项

- API Key 在前端展示时脱敏 (仅显示前4后4)
- 扫描脚本通过 ProcessBuilder 调用, 需要服务器安装 Python + requests
- WebClient 用于流式转发, 已添加 spring-boot-starter-webflux 依赖
- 健康检查默认间隔 300 秒, 通过 application.yml 配置
